I'm **Skryvon** — a full-stack software architect who operates at the intersection of **systems engineering**, **visual design**, and **security research**. My work spans from low-level kernel modules and reverse engineering binaries, to building pixel-perfect, animated interfaces that users actually enjoy.

<summary><b>💡 My Open Source Philosophy</b></summary>

<br/>

I approach open-source contributions with three principles:

1. **Don't ship ugly code.** If a project's codebase is worth using, it's worth contributing clean, well-documented changes to. PRs with zero comments or test coverage don't get my signature.

2. **Prioritize the DX.** Developer experience matters as much as performance. A tool that's fast but painful to use will be abandoned. I focus on ergonomic APIs, clear error messages, and sensible defaults.

3. **Maintain what you publish.** Nothing kills trust faster than abandoned packages. I commit to maintaining projects I author with release notes, changelogs, and security patches.

## 🧭 Developer Philosophy

<table>
<tr>
<td width="50%">

### 🏛️ Code Principles

> **"Write code as if the next person to maintain it is a sleep-deprived version of yourself."**

- **Readability first.** Performance optimizations belong in profilers, not first drafts.
- **Test edge cases.** Happy path code is incomplete code.
- **Abstract deliberately.** Premature abstraction is as dangerous as none.
- **Document as you go.** Comments explain *why*, code explains *what*.
- **Security by design.** Tack-on security is never real security.

</td>
<td width="50%">

### 🎨 Design Principles

> **"Good design is invisible. Great design is unforgettable."**

- **Hierarchy guides attention.** Every element should earn its space.
- **Color communicates.** Never decorate — always communicate.
- **Whitespace is not empty.** It's the structure that holds everything else.
- **Design for failure.** Error states and empty states are part of the design.
- **Prototype before pixel-pushing.** Validate concepts before polishing.

</td>
</tr>
</table>
